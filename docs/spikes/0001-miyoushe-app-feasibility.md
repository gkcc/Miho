# 米游社 APP 实际操作可行性探针

状态：待人工验证  
类型：只读 feasibility spike / 人工 checklist  
边界：不是 MVP 功能，不实现采集，不自动登录，不读取 cookie/token，不抓包，不写正式数据库。

## 目标

验证米游社 APP 中以下路径是否具备后续半自动采集价值：

* 我的；
* 对应游戏；
* 战绩统计；
* 角色概览；
* 角色详情页。

本 spike 只回答“这条实际操作路径是否值得后续研究”，不把 APP 自动点击纳入 MVP。重点观察页面字段、可重复性、UI Automation 可见性、WebView 可能性，以及截图/OCR 是否真的必要。

## 非目标

本 spike 明确不是为了：

* 自动登录；
* 自动点击完整流程；
* 抓包；
* 读取 cookie/token；
* 绕过风控；
* 自动 OCR 入库；
* 写正式数据库；
* 控制游戏客户端。

额外限制：

* 不读取本机 APP 数据目录。
* 不读取浏览器 profile 或 APP profile。
* 不保存真实账号标识。
* 不把截图、OCR 文本或人工记录直接作为正式练度数据。
* 不将结果写入 SQLite。

## 下一步动作

根据人工检查结论选择下一步：

* 如果 UIA 可读，下一步做只读 UIA prototype；prototype 只能读窗口和文本可见性，不写正式数据库，不接 cookie/token。
* 如果 WebView 可验证，下一步做 `saved_html` / `browser_page` adapter 研究；必须先定义隐私、登录边界和脱敏策略。
* 如果只能截图，下一步只做人工截图字段对照，不做自动入库；OCR 只能作为 P2/P3 候选并必须人工确认。
* 如果路径不稳定，回到手动 JSON 导入。

建议后续任务名称：


无论选择哪条下一步，都必须继续遵守：

* 不自动登录；
* 不读取 cookie/token；
* 不抓包；
* 不绕过验证码或风控；
* 不控制游戏客户端；
* 不把 OCR 结果自动写入正式数据库。
