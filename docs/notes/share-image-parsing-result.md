# 米游社官方分享图解析结果记录

状态：P0.6 probe 结果记录
日期：2026-06-27

## 结论

官方分享图值得继续作为 P1 数据源候选，但当前仍只能作为 probe 和字段抽取 prototype 输入，不能直接写正式数据库。

当前仓库已有历史解析结果显示：

* 布局：`zzz-agent-card`
* OCR 引擎：`tesseract`
* 覆盖等级：`medium`
* 旧覆盖口径：matched 31，missing 16

该结果说明固定版面裁剪和数字字段识别路线成立。中文字段质量仍取决于 OCR backend 和语言包，后续应优先验证 PaddleOCR。

## 字段覆盖情况

当前分享图路线预计覆盖：

* 角色名、等级、Rank。
* 核心数值：生命值、攻击力、防御力、冲击力、暴击率、暴击伤害、异常掌控、异常精通、贯穿力、能量自动累积、属性伤害加成。
* 六个技能等级。
* 音擎名称、等级、评级。
* 六个驱动盘的套装、等级、主词条、副词条。

## 稳定字段

较稳定：

* 数字字段，尤其是等级、生命值、攻击力、防御力、暴击率、暴击伤害。
* 固定区域内的技能等级。
* 图片尺寸一致时的区域裁剪。

仍不稳定：

* 中文角色名、音擎名、驱动盘套装名。
* 驱动盘副词条中标签和值的同行匹配。
* OCR 对 `LV.60`、`S/A/B/C` 等短文本的误读。
* 不同导出图尺寸或 UI 改版后的裁剪比例。

## 是否值得作为 P1 数据源

推荐等级：中高。

理由：

* 官方分享图由用户手动导出，可复现、可回放、离线处理，不需要读取 cookie/token。
* 字段覆盖已经接近 ADR-0003 的角色练度快照草案。
* 即使中文 OCR 不稳定，固定区域数字抽取也能先支撑人工确认页和 fixture 回放。

限制：

* OCR 结果必须人工确认后才能进入正式数据库。
* P0.6 仍不是正式采集器。
* 批量化前需要截图坐标、窗口尺寸和分享按钮位置 probe。

## 下一步

1. 使用 `--engine auto` 优先验证 PaddleOCR。
2. 为 `extracted_draft` 固定 JSON schema。
3. 用窗口截图和相对坐标 dry-run probe 标定官方分享按钮位置。
4. 后续只在官方 UI、已登录本地 APP、无账号密码输入、无 token 读取、无抓包的边界内评估 RPA。
