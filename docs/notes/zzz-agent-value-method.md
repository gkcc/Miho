# 绝区零 Box 代理人价值判断方法

本文记录新的产品主线：不再优先做完整练度工具，而是先做“当前 box 代理人价值判断”。目标是把公开高难统计、公开 Tier 信息、官方 box 图识别结果和可解释评分器结合起来，给出：

- 当前 box 内代理人价值排序；
- 式舆防卫战与危局强袭战配队建议；
- 哪些角色值得练，哪些只是过渡，哪些是未来观察；
- 为什么不为了过关去拉低价值角色。

## 输入分层

### 用户本地输入

来源是用户显式提供的官方 ZZZ box 总览图，或人工确认后的脱敏 roster JSON。

当前 box 图只能可靠支持：

- 已拥有代理人；
- 粗等级；
- 影画数字；
- 稀有度；
- 属性与特性图标；
- 识别置信度与人工确认状态。

当前 box 图不能可靠支持：

- 音擎；
- 驱动盘；
- 技能等级；
- 邦布；
- 资源库存；
- 角色是否毕业。

所以报告必须标注：基于 roster 图的建议最多是“中等置信度的配队与价值草案”，不是毕业度判断。

### 公开 Meta 数据

首批公开数据源：

- Prydwen ZZZ Tier List；
- Prydwen ZZZ Shiyu Defense；
- Prydwen ZZZ Deadly Assault；
- Prydwen ZZZ Banners；
- ShiyuStats / Hugging Face processed dataset，作为后续更稳的结构化 fallback。

Prydwen 页面与 API 只作为公开社区统计信号，不读取账号，不使用 cookie/token，不绕过反爬，不高频请求。

每份公开数据都必须带：

- `source_ref`
- `captured_at` 或 `generated_at`
- `content_sha256`
- `phase`
- `mode`
- `sample_size`
- `trust_level`
- `freshness`

## 公开数据解释边界

Appearance rate 不是持有率，也不是抽取价值。

Endgame 分数与队伍使用率是社区提交样本，会受以下因素影响：

- 样本玩家偏强；
- 新角色热度；
- 当期首领机制；
- 队伍是否排除了高影 S 级角色；
- 是否缺少 ownership rate；
- Prydwen 页面口径与 ShiyuStats processed 数据口径可能不完全一致。

因此公开数据只参与评分，不直接覆盖本地 box 事实。

## 双分数：现实价值与潜力价值

每个代理人同时输出两个价值：

```text
现实价值：现在这个账号练它是否能提高终局结果。
潜力价值：如果获得未来配件或补齐队友，它是否值得保留观察。
```

### 现实价值

现实价值用于决定“现在练不练”。

建议权重：

| 维度 | 权重 | 说明 |
| --- | ---: | --- |
| 当前绝对强度 | 20 | 单体机制、0 影可用性、专武依赖度 |
| 当前 endgame 表现 | 20 | 近期式舆/危局出场率、分数、队伍上限 |
| 当前 box 队伍生态 | 25 | 是否已有关键队友，是否能组成完整队伍 |
| 培养性价比 | 15 | 从当前等级到可用线的成本；成本只影响优先级和时间，不应把高价值低等级角色直接打成低价值 |
| 保值与泛用性 | 15 | 是否跨期、跨首领、跨属性环境可用 |
| 风险惩罚 | -5 到 -20 | 缺关键队友、机制太窄、和强队抢人、投入后收益低 |

### 潜力价值

潜力价值用于决定“要不要观察或等配件”。

建议权重：

| 维度 | 权重 | 说明 |
| --- | ---: | --- |
| 机制底子 | 25 | 是否有独特乘区、异常/击破/辅助能力 |
| 未来配件敏感度 | 25 | 缺一个关键队友后是否会质变 |
| 已知售后空间 | 20 | 已上线或官方高置信信息显示的同体系角色/驱动/音擎 |
| 低替代性 | 15 | 当前 box 内是否难以替代 |
| 沉没风险 | 15 | 提前练是否浪费资源 |

输出示例：

```text
现实价值：B+
潜力价值：A
当前建议：暂缓培养
原因：机制底子强，但当前 box 缺关键异常/支援配件，成队完整度只有 55%。
触发条件：抽到某类支援或异常队友后重新评估。
```

## Box 内 Tier

不要直接照搬全网 T0/T1。账号内 Tier 需要由本地 box 与公开数据共同决定：

```text
S：当前 box 已能成队，投入后显著改善式舆/危局结果。
A：当前可用，缺少少量配件或练度，收益明确。
B：机制或公开数据不错，但当前 box 缺关键队友，暂缓。
C：特定场景可用，或被 box 内已有高价值角色替代。
D：投入后仍不能改善终局结果，默认禁止主动推荐培养。
```

## 队伍匹配

每个目标先生成候选队伍，再做全局分配，不用贪心。

队伍评分：

```text
team_score =
  role_coverage
+ synergy_score
+ endgame_mechanic_fit
+ element_or_weakness_fit
+ build_readiness
+ mindscape_bonus
+ public_usage_signal
- investment_cost
- conflict_penalty
- uncertainty_penalty
```

式舆防卫战更重：

- 多队稳定；
- 属性覆盖；
- 持续输出；
- 队伍完整度；
- 不互相抢核心队友。

危局强袭战更重：

- 首领机制；
- 爆发窗口；
- 击破/异常节奏；
- 队伍上限；
- 当期 boss 分数表现。

## 角色冲突处理

如果多个目标争抢同一角色：

1. 为每个目标生成 Top N 候选队伍。
2. 约束同一角色同一轮最多使用一次，除非目标配置允许复用。
3. 求最大总分组合。
4. 如果无可行解，允许软冲突但给巨大惩罚。
5. 输出机会成本：角色给 A 比给 B 多提升多少分。

冲突报告示例：

```text
争抢角色：某核心支援
竞争目标：式舆上半、危局首领 2
推荐分配：给危局首领 2
原因：该目标缺少替代支援，失去该角色后队伍从可用降为不可用。
替代方案：式舆上半改用候选队 B，总分下降 8.5。
置信度：中
```

## 禁止为了过关拉垃圾角色

硬门槛：

```text
代理人现实价值 < C+ 且潜力价值不高：默认不推荐培养。
队伍完整度 < 65%：不推荐作为主力队。
培养后无法改变终局结果：不推荐。
需要高成本但只提升小场景：不推荐。
和另一队抢关键角色导致总收益下降：不推荐。
```

低等级不等于低价值。如果角色本身高 Tier、高保值、公开高难表现好，且能组成强队，应该输出“需要投入资源拉起”，而不是“观察”。真正需要拦的是低 Tier、下坡、缺生态、公开使用弱、投入后也不能改善结果的角色。

如果必须临时使用低价值角色，只能输出“过渡使用”，不能输出“值得培养”：

```text
可临时使用，但只建议拉到最低可用线：
- 等级到关键突破；
- 核心技到关键节点；
- 音擎使用已有替代；
- 驱动盘只配主词条，不刷副词条；
- 一旦获得更高价值配件，停止投入。
```

## Banner 与未来配件

Prydwen banner 页可以作为公开汇总信号，用于判断：

- 某角色是否近期可获得；
- 某体系是否有官方已公开售后；
- 当前缺失配件是否有可规划窗口。

但 banner 页不能把未拥有角色算入当前可用队伍。它只影响：

- 潜力价值；
- 未来观察名单；
- 抽取前置问题；
- 资源预留提醒。

来源置信度建议：

| 信号 | 置信度 | 用途 |
| --- | ---: | --- |
| 已拥有角色 | 100% | 现实价值与队伍匹配 |
| 已上线但未拥有 | 70% | 潜力价值与抽取问题 |
| 官方公告/直播已公开 | 50% | 未来观察 |
| Prydwen banner 汇总但未回指官方 | 40% | 仅提示核对 |
| 社区推测/卫星 | 20% | 不进入正式推荐 |
| 纯模型猜测 | 0% | 不使用 |

## 大模型角色

大模型只负责：

- 整理解释；
- 归纳风险；
- 把结构化证据转成可读建议；
- 在低置信度处提示人工确认。

大模型不负责：

- 凭空定义 Tier；
- 覆盖评分器；
- 把未拥有角色算进当前队伍；
- 把 appearance rate 解释成抽取价值；
- 为了凑队建议重投入低价值角色。
